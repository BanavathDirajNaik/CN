📄 PROJECT DOCUMENT
Real-Time Network Data Analyzer using Grafana
1. Objective

The objective of this project is to design and implement a real-time Network Data Analyzer that monitors all running applications in a system and identifies:

Which application (process) is using the network
How much data is being transmitted (TX) and received (RX)
Where the data is being sent (destination IP/port)
Real-time visualization of all these metrics using
Grafana
2. Problem Statement

Traditional network monitoring tools only show overall bandwidth usage or interface-level statistics.

They do NOT provide:

Application-level visibility
Process-level network usage
Real-time traffic flow analysis

This project solves that by building a system that:

Tracks network activity at the process level and visualizes it in real time.

3. System Architecture

The system follows a pipeline-based architecture:

Applications (Chrome, curl, iperf)
            ↓
Process Monitoring (PID tracking)
            ↓
Connection Mapping (ss command)
            ↓
Python Exporter (data collection)
            ↓
Prometheus (metrics scraping)
            ↓
Grafana Dashboard (visualization)
4. Technologies Used
Component	Tool	Purpose
Programming	Python	Data collection & exporter
Monitoring	Prometheus	Metrics scraping
Visualization	Grafana	Dashboard display
OS Tools	Linux (ss, /proc)	Process & network mapping
Library	psutil	Process monitoring
5. Methodology
Step 1: Process Identification
All running processes are identified using Python (psutil)
Each process is mapped with:
PID
Application name
Step 2: Network Connection Mapping
The command below is used:
ss -tunp
It provides:
PID
Source IP
Destination IP
Port number
Step 3: RX / TX Data Collection
For each process:
RX (Received bytes)
TX (Transmitted bytes)
Data is collected at 1-second intervals
Step 4: Exporting Metrics
A Python exporter sends metrics to Prometheus:

Example metrics:

app_rx_bytes
app_tx_bytes
app_connections
Step 5: Visualization
Data is visualized using Grafana dashboards:
Real-time graphs
Tables
Top applications

RX/TX accuracy depends on OS-level data
Android integration requires additional APIs
